# Описание проекта
На втором курсе во время участия в хакатоне нам срочно потребовались frontend разработчик, который смог бы создать сайт с адаптивной версткой.

Мы искали его:

- Среди друзей и их друзей
- Среди одногруппников
- В беседах потока
- В приложении для студентов
- На сайте самого хакатона…

Поэтому мы предлагаем наше собственное решение - TeamBuilder.

Мы хотим создать сервис для поиска участников в ваших проектах

○ Создавайте собственные проекты

○ Настраивайте теги

○ Присоединяйтесь к интересным вам проектам

○ Находите людей для хакатонов или лабораторных

Мы изучили аналоги:

- хакатоны.рф
    - Поиск существующих хакатонов
- My ISU
    - Приложение для студентов ИТМО, где запрятана функция поиска команды

- TeamBuilder
    - Поиск интересующих проектов по тегам
    - Просмотр запросов по тегам и описанию

# Пользовательские сценарии
Для того чтобы лучше разобраться в том, какие пользовательские сценарии нам нужны, мы составили user flow для нашего сервиса.

Мы видим его таким:

![](media\user-flow.png)

Основываясь на этой диаграмме, мы можем разделить функционал нашего приложения на 3 основные группы:

1. Аутентификация и регистрация пользователей
1. Создание и поиск проектов
1. Получение уведомлений

Стоит уточнить, что п.2 и п.3 разделены, т.к. при совершении одних и тех же действий у разных групп пользователей (участники и организаторы) происходят разные события, поэтому при таком разделении разработчику куда проще понять логику работы.
## Аутентификация и регистрация пользователей

| User story name | User story description | Notes |
| --- | --- | --- |
| Авторизация | Пользователь хочет войти в свой аккаунт в системе | Добавить кнопку для авторизации, Создать форму для ввода логина и пароля, Добавить валидацию на текстовые поля |
| Регистрация | Пользователь хочет зарегистрироваться в системе | Добавить кнопку для регистрации, Создать форму для ввода логина и пароля, Добавить валидацию на текстовые поля |
| Просмотр профиля | Пользователь хочет посмотреть данные своего профиля, свои проекты и теги | Добавить кнопку для перехода к данным аккаунта, Создать страницу с редактируемыми полями для ввода информации об аккаунте, предлагаемыми тегами и карточками своих проектах, Добавить валидацию на текстовые поля и дату |
| Обновление данных профиля | Пользователь хочет обновить данные в профиле | Добавить редактируемую форму с данными пользователя в профиле аккаунта, Добавить кнопку для обновления данных пользователя, Добавить валидацию на текстовые поля и дату |
| Выйти из аккаунта | Пользователь хочет выйти из своего аккаунта | Добавить кнопку для выхода из аккаунта |

## Создание и поиск проектов

| User story name | User story description | Notes |
| --- | --- | --- |
| Создание проекта | Пользователь хочет создать новый проект для поиска разработчиков | Добавить кнопку создания нового проекта, Создать форму для заполнения информации о проекте, Добавить возможность выбора тегов определенной тематики, Добавить уведомления о создании проекта и получении заявок от новых пользователей, Добавить обработку ошибок |
| Подать заявку в проект | Пользователь хочет подать заявку в понравившийся ему проект | Добавить кнопку подачи заявки в проект, Настроить уведомления об принятии или отклонении заявок, Добавить обработку ошибок |
| Поиск проекта по названию | Пользователь хочет найти проект по определенному названия используя поисковую строку | Добавить поисковую строку, Настроить фильтрацию проектов и описаний, Добавить обработку ошибок |
| Поиск проекта по тегам | Пользователь хочет найти проект по определенной теме используя список тегов | Добавить к поисковой строке основные теги, Добавить дополнительные теги для конкретизации проектов, Добавить возможность выбора / удаления тегов для фильтрации |

## Получение уведомлений

| User story name | User story description | Notes |
| --- | --- | --- |
| Присоединиться к новому проекту | Пользователь хочет присоединиться к новому проекту | При присоединении к проекту организатору приходит уведомление, В специальной форме он видит участника и его навыки, Организатор принимает решение по поводу добавления учащегося |
| Организатор принял решение по добавлению нового учащегося | Организатор видит основные данные о пользователе и решает, добавлять его или нет | После решения организатора участник получает уведомление, В специальной форме он видит, что его заявку приняли или отклонили, В своих проектах у него стал отображаться новый проект |


# Дерево задач
Составив пользовательские сценарии, мы можем составить примерное дерево задач:

- Проект TeamBuilder
    - Первый спринт
        - Создать бэкенд
            - Добавить Spring Boot Application в проект
            - Добавить POJO и настроить БД
            - Добавить Spring Security
            - Добавить авторизацию
            - Добавить сервисы для создания проектов
        - Создать фронтенд
            - Добавить хэдер
            - Добавить страницу с профилем
    - Второй спринт
        - Бэкенд
            - Добавить поиск по проектам и настроить репозитории для БД
            - Добавить теги и настроить репозитории для БД
        - Фронтенд
            - Добавить теги в недостающие места
            - Добавить поиск
    - Третий спринт
        - Бэкенд
            - Добавить уведомления
            - Добавить второй сервис с сообщениями
        - Фронтенд
            - Добавить страницу для уведомлений
            - Добавить теги в профиль и проектную страницу
